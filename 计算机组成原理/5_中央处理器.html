5 中央处理器 cpu

  • 计算机的核心
  • 包括运算器和控制器

5.1 中央处理器的功能和组成

CPU主要功能：

  1. 控制指令的执行顺序
  2. 产生控制信号控制部件工作
  3. 控制各部操作的时序
  4. 数据处理：算术和逻辑运算

CPU的组成从硬件设计的角度：

CPU寄存器：通用、专用

  1. 通用寄存器：
  2. 专用寄存器： 至少5个：
    • 程序计数器PC/指令计数器IR/存储器地址寄存器MAR/存储器数据寄存器MDR/状态标志寄存器PSWR

PC->MAR->主存（地址线） 主存->MAR(数据线：间接寻址) 主存->MDR(数据线：数据地址) MDR->IR() 指令=操作码+地址码

状态标志寄存器PSWR: ??

5.2 控制器的组成和实现方法

控制器是指挥中心

  1. 指令部件
    1. 程序计数器PC
    2. 指令寄存器IR
    3. 指令译码器ID
    4. 地址形成部件
  2. 时序部件
    1. 脉冲源
    2. 启停控制逻辑
    3. 节拍信号发生器
  3. 微操作信号发生器
  4. 中断控制逻辑

控制器的输入是机器指令代码 输出是微操作控制信号

常见的微操作信号发生器实现方式：

  • 组内逻辑器
  • 存储逻辑型
  • 组合逻辑与

控制器的硬件实现方法：

  1. 存储
  2. 存储逻辑性：每条指令

5.3 时序系统与控制方式

时序系统是控制器的心脏

  1. 指令周期和机器周期 指令周期是指取指令、分析指令到执行完该指令所需的全部时间。 指令周期=i*机器周期

  2. 节拍 一个机器周期内，要完成若干个微操作 节拍的宽度取决于CPU完成一次基本操作的时间 节拍选取一般有几种方法：

    1. 统一节拍法：定长CPU周期
    2. 分散节拍法：不定长CPU周期
    3. 延长节拍法：
    4. 时钟周期插入

  3. 工作脉冲

  4. 多级时序系统

CPU各部件之间的控制方式

  1. 同步控制方式：集中控制方式或中央控制方式
  2. 异步控制方式：分散控制方式或局部控制方式

指令执行的基本过程

  1. 取指令阶段：将线性指令从主存中取出来并送到指令寄存器中去，所有指令均有（公共操作）
    1. PC->MAR\
    2. Read\
    3. M(MAR)->MDR\
    4. (MDR)->IR\
    5. (PC)+1->PC
  2. 分析取数阶段 指令译码器ID
  3. 执行阶段

指令的微操作序列

不同的数据通路就有不同的微操作序列

  1. 加法指令ADD @R0,R1

    取指令阶段的公共操作： 1.

    1. 取数周期： $R_{0out}$和$MAR_{in}$有效

    2. 执行周期： 转移指令

微程序控制原理

微程序设计技术的实质是将程序设计和存储技术相结合 基本术语

  1. 微命令和微操作 微命令和微操作是一一对应的。微命令是微操作的控制信号
  2. 微指令、微地址
  3. 微周期
  4. 微程序：一条机器指令对应于一段微程序
  • 微程序和程序两个不同的概念

一条微指令通常至少包含两大部分信息：

  1. 操作控制字段
  2. 顺序控制字段

微指令编码法

  1. 直接控制法（不译码法） 1：有效 0：无效 并行性强，快，指令字太长
  2. 最短编码法 微指令字长最短 L>=log2N
  3. 字段编码法
